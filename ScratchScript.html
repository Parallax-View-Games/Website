Scratch Script

What is it?

Scratch script is a custom programming language that I made. The interpreter is themed like the scratch editor because this language is meant to be like a bridge for scratch users. They learn this so that learning other programming languages will be easier. Pieces of code are called commands. I will refer to them as commands in the syntax section. This is because the language was originally called "Command Code" and pieces code look like command prompts. In the next section, I will include some rules for the syntax.

Syntax: basic rules

All commands start with a forward slash and all commands end with brackets. strings don't require quotation marks. for example, instead of this, "/costume.set("costume name")", you would need to use this, "/costume.set(costume name)".

Syntax: basic commands

Most commands follow a three part rule. They start with the type of command followed by a dot then the main command. After that, any value or string needed is placed in brackets at the end of the command. Commands that don't require a string or value still need to have brackets. for example, the code to hide a sprite doesn't have any value but it still has brackets; "/costume.hide()".

Syntax: running code

All code needs to be within one of these 2 commands, "//start{}" and "//forever{}". The code needs to be within the curly brackets, any code outside of these brackets will not run. Code inside the start command will run once as soon as the project runs. Code inside the forever command will loop until the project stops. Also, placing this, "!", before the brackets in a forever command will make the interpreter loop through all of the code in a single tick. This makes projects run smoother but wait commands do not work and it speeds up the project significantly which isn't useful in some situations.

Syntax: IF commands

IF checks are very different from other commands. they start normally with "/if.". then there is the condition which has to be placed in square brackets. finally, the code that the if check runs is placed in the normal brackets at the end. an if check would usaully look like this; "/if.[d.pressed](/move.right(10))". Also, placing this, "!" , at the start of a condition will turn it into an "if not" check. IMPORTANT: Nested if commands DO NOT WORK! Although they technically work, it causes the interpreter to lag and sometimes crash.

Syntax: Looping commands

Loops are very similar to If commands, there are only 2 differences. First of all, the "/if" is replaced with "/repeat". The number of times that the loop will run will replace the condition in the square brackets. A loop command would look something like this; "/repeat.[10](/move.up(1))".

Syntax: Variables

Variables need to be created before they can be used. They can be created using this command; "/var.create(variable name)". After they're created, you can set a variable to any value or string using this command; "/var.set_variable(text or value)". You can also add values to variables using this command; "/var.add_variable(value)".

Syntax: Reading values

If you need the code to read the value of a variable or get info such as the sprites X position, you'll need to use reporters. Any text inside of double quotation marks is a reporter. The interpreter will replace reporters with the value they represent.

Syntax: Defining scripts

If you need to write the same code multiple times, it is best to define it. To define code, you need to use this command; "/define.name(code)". To run defined code, you use this command; "/run.name()". It is possible to define code during runtime however, it is recommended that all definitions are placed into the //start command, not the //forever command.
